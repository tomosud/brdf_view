# BRDF Explorer Web

Disney BRDF Explorer の WebGL2 移植実験です。元実装の `.brdf` shader 断片を JavaScript へ置き換えず、GLSL ES 3.00 の shader template へ注入して GPU 上で評価する方針を取っています。

- Web app: https://tomosud.github.io/brdf_view/
- Windows comparison runtime: https://github.com/tomosud/brdf_view/releases/tag/brdf-runtime-v0.1.0
- Research notes: [docs/brdf_explorer_web_port_research.md](docs/brdf_explorer_web_port_research.md)
- Windows exe build notes: [docs/windows_exe_build.md](docs/windows_exe_build.md)

## Reference

This project is based on Disney's BRDF Explorer source in `sample/brdf-main`.

The Web implementation is intended to reproduce the behavior of the original desktop tool, but it is not a direct Qt/OpenGL port. The current comparison target is a locally rebuilt Windows x64 runtime, distributed as `brdf-runtime.zip` in the release linked above.

## Current Implementation

The Web app lives under [web/](web/). It uses Vite, TypeScript, and WebGL2.

Implemented:

- `.brdf` parser for analytic BRDFs
- Runtime parameter UI for float, bool, and color parameters
- GLSL template injection similar to `BRDFBase::loadShaderFromFile`
- Bundled analytic BRDF loading, including `lambert.brdf` and `disney.brdf`
- User file loading for `.brdf` and MERL `.binary`
- 3D Plot
- Polar Plot
- Cartesian `Theta V` Plot
- Lit Sphere
- Image Slice
- Lit Object with equirectangular HDRI background and cosine-sampled IBL
- GitHub Pages deployment through GitHub Actions

Not yet equivalent to the original:

- `.bparam` load/save
- MIT CSAIL anisotropic `.dat`
- `.tif` image-slice file loading
- `Theta H`, `Theta D`, and `Albedo` Cartesian views
- original Ptex `.penv` probe loading
- OBJ model loading in Lit Object
- original IBL importance sampling / BRDF IS / MIS modes
- progressive accumulation matching the desktop IBL renderer

## Local Development

Install and run:

```powershell
cd web
npm ci
npm run dev
```

Build:

```powershell
cd web
npm run build
```

Preview the built GitHub Pages output locally:

```powershell
.\serve_pages_local.bat
```

This serves `web/dist` at:

```text
http://localhost:4173/brdf_view/
```

## GitHub Pages

The repository uses [`.github/workflows/pages.yml`](.github/workflows/pages.yml).

On push to `main`, GitHub Actions runs:

```text
cd web
npm ci
npm run build
```

Then it deploys `web/dist` to GitHub Pages. The Vite base path is `/brdf_view/`, matching the project page URL.

## Assets

Tracked assets:

- `assets/ibl.hdr`: default HDRI for the Lit Object view
- `web/public/brdfs/*.brdf`: bundled analytic BRDFs
- `web/public/shaderTemplates/*`: WebGL2 shader templates
- `web/public/LICENSE`, `LICENSE-BINARY`, `README`

Generated or local-only assets:

- `web/public/environments/`: generated by `npm run copy-assets`
- `web/public/measured/`: local sample MERL assets when available
- `web/dist/`: generated Vite build output

The "Load sample MERL" button depends on `web/public/measured/gold-metallic-paint3.binary`. That file is large/licensed local data and is not guaranteed to be available on GitHub Pages unless explicitly published.

## Differences From The Original Renderer

The following differences can affect rendered results.

### Shader Language And Injection

The original uses desktop GLSL 410. This project uses WebGL2 / GLSL ES 3.00.

The implementation preserves `.brdf` shader bodies as much as possible, but it includes a lexical integer-literal promotion pass to make bundled desktop GLSL snippets compile under GLSL ES. This is intentionally narrow. User `.brdf` files containing integer loops, array indexing tricks, texture fetch indices, or other GLSL 410-specific constructs may fail to compile or render differently.

### IBL / Lit Object

This is the largest visual difference.

The original `IBLWidget` loads Ptex `.penv` cube probes, creates cubemap and probability textures, and progressively accumulates samples using `stepSize = 271` and `totalSamples = stepSize * 15`. It exposes `No IBL`, `IBL: No IS`, `IBL: IBL IS`, and UI entries for `BRDF IS` / `MIS`.

The Web version currently:

- loads a normal equirectangular HDRI (`assets/ibl.hdr`)
- samples it as a 2D texture, not a cubemap
- uses cosine-weighted Monte Carlo sampling
- does not build the original environment CDF/probability textures
- does not implement `IBL: IBL IS`, `BRDF IS`, or `MIS`
- does not progressively accumulate into a float FBO
- always renders a generated sphere, not an arbitrary OBJ

As a result, Lit Object brightness, noise, highlight placement, environment orientation, and convergence behavior can differ substantially from the Windows comparison exe.

### Plot Lines

The desktop app uses geometry shaders such as `brdftemplatePlot.geom` to expand `GL_LINE_STRIP_ADJACENCY` into antialiased thick lines.

WebGL2 has no geometry shaders, so this project expands plot lines in vertex shaders using `gl_VertexID`. The miter math is close, but the original's extra gap-fill behavior at sharp corners is not implemented. Very sharp BRDF curves can show slightly different joins or gaps.

### 3D Plot

The geodesic hemisphere subdivision matches the original approach, but the Web view currently omits the original gray floor plane. Axis/direction guide drawing is also simpler. The BRDF lobe shape should be the main comparable element; surrounding guides may not match pixel-for-pixel.

### Cartesian Views

Only `Theta V` is implemented. The original has `Theta V`, `Theta H`, `Theta D`, and `Albedo`. The Web version also uses simplified grid/axis drawing and does not render the original bitmap text labels.

### Image Slice

The Web Image Slice evaluates analytic or MERL BRDFs over `(thetaH, thetaD)` and supports `phiD`, gamma, exposure, squared thetaH, and chroma. It does not implement real `.tif` file loading. The original source's `.tif` path is also limited in practice, but the desktop UI has brightness handling and frame/guide drawing that are not fully duplicated here.

### MERL Measured BRDF

MERL `.binary` parsing follows the original data layout and RGB scale constants, replacing desktop `samplerBuffer` with a packed `R32F` 2D texture. Results should be close for standard MERL files, but precision and texture-size behavior depend on browser/GPU WebGL2 support.

### Browser/GPU Constraints

The app requires WebGL2 and high precision float support. HDRI filtering uses `OES_texture_float_linear` when available and falls back to nearest filtering otherwise, which can visibly affect the Lit Object environment.
